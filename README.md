# WinRehome

一个面向 Windows 的迁移备份工具，目标是在尽可能节省备份空间的前提下，保留真正有迁移价值的个人数据。

它不做整盘镜像，也不试图把整台系统原样搬走，而是把真正值得迁移的内容整理成一个更克制、更易恢复的单文件备份。

## 为什么是 WinRehome

- 已安装软件只保留清单，不浪费空间备份安装包和程序本体
- 便携软件按候选项识别，并和个人文件一起打进单一归档
- 个人文件按迁移价值筛选，默认排除缓存、临时文件、日志、构建产物等可重建内容
- 归档可由应用自身读取和恢复，不依赖额外脚本

## 主要能力

- 扫描 Windows 已安装软件并生成清单预览
- 识别便携软件候选并给出默认选择建议
- 收集高价值用户目录并估算文件数与体积
- 生成单文件 `.wrh` 归档
- 读取 `.wrh` 归档并恢复到目标目录
- 保存最近一次选择、归档路径和恢复目录，减少重复配置
- 按类别恢复用户文件或便携软件，而不是每次都整包恢复
- 支持按具体根目录勾选恢复内容，并可一键全选或清空
- 恢复前会显示已选根目录、预计文件数和预计恢复大小
- 可先校验归档完整性，再执行恢复
- 遇到目标位置已有文件时，默认拒绝覆盖，也可显式选择跳过已有文件
- 在恢复时校验文件大小与 CRC，避免无提示写入损坏内容
- GitHub Release 自动构建并上传 Windows `exe`

## 工作方式

WinRehome 当前采用“扫描 -> 审查计划 -> 生成归档 -> 恢复归档”的流程：

1. 扫描已安装软件、便携软件候选和高价值个人文件目录
2. 让用户审查默认选择，而不是做黑盒式全自动打包
3. 生成单文件 `.wrh` 归档，内部保存文件数据、安装软件清单和归档元数据
4. 在新系统或目标目录中读取 `.wrh` 并执行恢复

## 归档格式

当前原型使用自定义 `.wrh` 格式：

- `WRH1` 文件头
- 每个文件独立压缩存储
- manifest 尾部索引，记录源路径、归档路径、偏移、大小和 CRC32

这让 WinRehome 可以在不依赖镜像备份的前提下，专注于更节省空间的迁移型备份。

## 本地开发

在仓库根目录执行：

```powershell
cargo check
cargo fmt
cargo test
cargo run
```

如果当前环境存在代理导致依赖下载失败，需要先清理代理环境变量再执行 `cargo` 相关命令。

## 发布产物

GitHub Release 发布后会自动触发工作流，在 Release 资产中上传：

- `WinRehome-vX.Y.Z-windows-x64.exe`
- `WinRehome-vX.Y.Z-windows-x64.exe.sha256`

## 技术栈

- Rust
- `eframe/egui`
- `winreg`
- `walkdir`
- `flate2`

## 适用场景

- 重装 Windows 前整理迁移数据
- 从旧电脑迁移到新电脑
- 需要保留便携软件和个人文件，但不想做整机镜像
- 希望备份结果尽量小、结构清晰、可恢复
